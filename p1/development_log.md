## [2026-03-09 19:17] 技术选型确定

- **前端框架**：选定 **React 18 + Next.js 14 (App Router)**
  - 理由：产品涉及大量状态管理（食材数组、模式开关、口味滑块），React 的生态和 Next.js 的 API Routes 可以同时覆盖前端和 Serverless 后端需求
- **样式方案**：选定 **TailwindCSS 3 + Framer Motion**
  - 理由：TailwindCSS 提供快速布局能力，Framer Motion 在 React 生态中动画支持最成熟，适合食材飞入、烹饪等复杂动画
- **状态管理**：选定 **Zustand**
  - 理由：轻量、无 boilerplate、TypeScript 友好，适合中等复杂度的状态（比 Redux 简洁）
- **国际化**：选定 **next-intl**
  - 理由：与 Next.js App Router 深度集成，支持中英双语切换
- **后端/AI 交互**：选定 **Next.js API Routes (Serverless)** + **LLM API**
  - 主选：OpenAI GPT-4o（创意能力强）
  - 备选：DeepSeek（降低成本）、Claude 3.5 Sonnet（Prompt 遵从性好）
- **部署方案**：选定 **Vercel**
  - 理由：与 Next.js 原生集成，免费托管，自动构建
- 输出文件：`tech_stack.md`
