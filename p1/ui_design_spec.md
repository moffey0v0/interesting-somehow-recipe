## 1. 设计理念

**核心关键词**：卡通 · 温馨 · 魔法 · 幽默  
**视觉隐喻**：一间整洁明亮的魔法厨房，木质台面上摆满了发光的炼金瓶罐，充满了「如果厨房和霍格沃茨魔药课融合」的感觉。

## 5. 组件交互逻辑

### 5.1 食材选择交互
1. 用户点击食材图标
2. 图标播放「弹跳」微动画（scale 1.0 → 1.2 → 1.0）
3. 食材图标的一个副本以抛物线轨迹飞向右侧食材槽
4. 食材槽中对应食材数量 +1，带有 `pop` 入场动画
5. 再次点击同一食材 → 数量继续 +1
6. 点击食材槽中的食材 → 数量 -1（若为 0 则移除），带 `fade out` 动画

### 5.2 烹饪触发交互
1. 锅处于「待机」状态时，轻微浮动 + 发光呼吸效果
2. 用户点击锅
3. **安全校验**（前端快速检查）：
   - ✅ 通过 → 锅开始「烹饪」动画（食材在锅里跳跃旋转，类似旷野之息）
   - ❌ 不通过 → 锅播放「冒黑烟 → 着火 → 爆炸💥」动画 → 弹出警告框
4. 烹饪动画作为 Loading 状态（约 3-8 秒，取决于 AI 响应时间）

### 5.3 结果展示交互
1. 烹饪完成 → 画面过渡到「出单台面」（场景切换动画）
2. 1-3 张小纸条从上方「飘落」到台面上
3. 每张小纸条可悬停查看概要（菜名 + 翻车率 + 食材摘要）
4. 点击某张纸条 → 展开为完整菜谱海报
5. 海报模式下提供「保存为图片」按钮

### 5.4 模式切换交互
- **生存模式 → 创造模式**：切换时，界面从冷调蓝绿色微变为暖调橙黄色
- **创造模式 → 生存模式**：反向变化
- 切换动画时长 300ms，使用 `ease-in-out`

## 7. 动效规范

| 动效 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 食材弹跳 | 200ms | spring(stiffness: 300) | 点击反馈 |
| 食材飞行 | 500ms | ease-out | 抛物线轨迹 |
| 锅呼吸光效 | 2000ms | ease-in-out | 循环 |
| 烹饪旋转 | 800ms/圈 | linear | 循环至 AI 响应 |
| 爆炸效果 | 1000ms | ease-out | 一次性 |
| 纸条飘落 | 600ms | spring(damping: 15) | 入场动画 |
| 场景切换 | 400ms | ease-in-out | 页面过渡 |
